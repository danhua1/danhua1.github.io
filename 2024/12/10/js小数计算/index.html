js经典问题0.1+0.2!=0.3

在 JavaScript 中，0.1 + 0.2 !== 0.3 的现象和 0.1 + 41.1 === 41.2 的现象，都是由于 浮点数精度 所引起的。

浮点数表示的基础

JavaScript 使用的是 IEEE 754 双精度浮点数标准，它在二进制下表示小数时，某些十进制小数（例如 0.1 和 0.2）无法精确表示，只能近似存储。这会引发计算中的精度误差。

1. 0.1 + 0.2 的问题
二进制表示：

0.1 转换为二进制后是一个无限循环小数：0.0001100110011...
0.2 转换为二进制后也是一个无限循环小数：0.001100110011...
由于计算机存储空间有限，它们都被截断为近似值。

相加后： 加法结果的近似值约为 0.30000000000000004，因此它不等于精确的 0.3。

2. 0.1 + 41.1 的情况
二进制表示：
41.1 的二进制表示与小数部分相对较短且更精确，因此误差较小。
相加后： 由于 0.1 的误差被小数部分的小权重所稀释，加法的结果仍然足够接近实际值，因此 0.1 + 41.1 被正确地舍入为 41.2。

总结
误差来源： 浮点数的二进制表示精度问题。
误差影响： 大数（如 41.1）相对小数（如 0.1）的误差权重较小，因此结果更接近真实值。

总结

方法 优势 适用场景
手动控制精度 简单直接 结果展示
整数运算 高效稳定 基础计算
高精度数学库 精确强大，功能丰富 高精度计算，如金融、科学计算等
避免直接比较小数 灵活处理浮点误差问题 判定结果是否接近
